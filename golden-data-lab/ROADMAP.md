# Golden Data Lab 학습 및 분석 로드맵

이 문서는 데이터 분석 실무 역량 강화를 위한 단계별 분석 학습 로드맵입니다.
기존 골든타임 프로젝트 데이터를 활용하여 **SQL 분석, Python EDA, Power BI 시각화, 통합 보고서 작성**까지의 전체 파이프라인을 경험합니다.

## 📂 단계별 상세 계획

### 1단계: SQL 분석 프로젝트 (3주)
* **경로**: `golden-data-lab/01_sql_public_health_analysis/`
* **목적**: 문법 암기 위주가 아닌, 질문 중심의 데이터 조회 능력 배양 (PostgreSQL 사용)
* **핵심 질문**:
  - 행정동별 취약인구는 얼마나 다른가?
  - 취약인구가 많은 지역은 어디인가?
  - 최근접 병원까지 거리가 긴 지역은 어디인가?
  - 취약인구도 많고 거리도 먼 지역은 어디인가?
  - 소아와 고령층의 우선지역은 어떻게 다른가?
  - 구·군별로 의료 접근성 격차가 얼마나 나는가?
  - 상위 10개 위험지역은 전체 위험의 몇 퍼센트를 차지하는가?
* **학습 문법**: `SELECT`, `WHERE`, `ORDER BY`, `GROUP BY`, `JOIN`, `CASE WHEN`, `CTE`, `WINDOW FUNCTION`, `VIEW`

### 2단계: Python EDA (3주)
* **경로**: `golden-data-lab/02_python_eda_public_health/`
* **목적**: SQL에서 도출한 결과를 Python(Pandas, Seaborn 등)으로 상세 검증 및 시각화
* **주요 작업**:
  - CSV 불러오기, 자료형/결측치/중복/이상치 확인 및 정제
  - 거리, 인구, VDI 분포 확인 및 상관관계 시각화
  - "이 이상치를 어떻게 처리할 것인가?", "평균값과 중앙값 중 무엇이 적절한가?" 등 통계적 판단 프로세스 학습

### 3단계: Power BI 대시보드 (2주)
* **경로**: `golden-data-lab/03_powerbi_public_health_dashboard/`
* **목적**: 의사결정자를 위해 핵심 KPI 및 시각 자료를 대시보드 형태로 구성
* **주요 구성**:
  - 전체 행정동 수, 고위험 행정동 수, 평균 최근접 병원 거리, 취약인구 합계
  - 구·군별 비교 차트, 소아/고령자 필터, 위험지역 상위 10개 테이블 및 지도 시각화
  - 핵심 가치: 의사결정을 지원하기 위한 비즈니스 관점의 대시보드 구성

### 4단계: 통합 분석 보고서 (1주)
* **경로**: `golden-data-lab/04_integrated_report/`
* **목적**: SQL, Python, Power BI를 거친 모든 분석 결과와 정책적 대안을 8~12페이지 내외의 실무형 보고서로 종합
* **보고서 구조**:
  1. 문제 정의
  2. 데이터 소개
  3. 데이터 품질 점검
  4. SQL 분석 결과
  5. Python EDA 결과
  6. Power BI 대시보드 소개
  7. 핵심 인사이트
  8. 분석의 한계점
  9. 정책 및 실무 활용 방안

### 5단계: 두 번째 주제로 반복 (차후 진행)
* **목적**: 첫 번째 프로젝트에서 익힌 역량을 바탕으로 새로운 도메인 데이터에 분석 프레임워크를 적용
* **추천 주제**: 대구 교통사고, 대구 인구감소와 고령화, 복지시설 접근성, 생활인구와 대중교통 등
