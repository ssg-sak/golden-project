# 백엔드 에이전트 규칙 (BACKEND_AGENTS.md)
Version: 1.0

이 규칙은 프로젝트의 백엔드 API 설계, 비즈니스 로직 구현 및 데이터베이스 상호작용을 담당하는 에이전트가 준수해야 하는 지침입니다.

## 1. Python 타입 힌팅 (Type Hinting) 의무화
- **정적 타입 선언**: 가독성과 조기 오류 감지를 위해 모든 백엔드 코드의 함수 정의, 변수 선언 시 타입 힌트를 명확히 명시합니다.
  - 예: `def get_locations(limit: int = 10) -> list[dict[str, Any]]:`
- **Pydantic 스키마 적극 활용**: FastAPI 엔드포인트의 요청(Request) 및 응답(Response) 모델 정의 시 Pydantic 스키마를 사용하여 데이터 유효성을 엄격하게 제어합니다.

## 2. API 설계 및 예외 처리 규칙
- **REST API 규칙 준수**: HTTP Method(GET, POST, PUT, DELETE)를 목적에 맞게 올바르게 사용하고, 엔드포인트 네이밍 규칙(복수형 명사 사용 등)을 일관되게 적용합니다.
- **적절한 HTTP Status 코드 사용**: 성공(200, 201), 클라이언트 에러(400, 401, 403, 404), 서버 에러(500) 등 상황에 최적화된 HTTP 상태 코드를 반환합니다.
- **강력한 예외 처리 및 검증**: 모든 외부 입력값에 대해 철저한 검증(Validation)을 수행하고, 예외 발생 시 구체적이고 안전한 에러 메시지와 함께 적절한 예외 응답을 처리합니다.

## 3. 데이터베이스 및 보안 규칙
- **SQL Injection 방지**: ORM(SQLAlchemy 등) 또는 파라미터화된 쿼리(Parameterized Query)를 필수로 사용하여 보안 취약점을 사전 예방합니다.
- **데이터베이스 정규화 및 PK/FK 관계**: 스키마 설계 시 정규화 원칙을 고려하고 기본키(PK)와 외래키(FK)를 명확히 설정합니다.
- **로컬 DB 마이그레이션**: 데이터 파이프라인(JSON 수정 등)에 변화가 생긴 뒤에는 반드시 로컬 DB 마이그레이션을 잊지 않고 동기화합니다.
- **인덱스(Index) 가이드**: 쿼리 성능 최적화를 위해 인덱스를 추가하는 경우, 추가 배경과 설계 이유를 주석이나 문서로 기록합니다.
