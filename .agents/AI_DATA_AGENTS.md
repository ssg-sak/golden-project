# AI & 데이터 에이전트 규칙 (AI_DATA_AGENTS.md)
Version: 1.0

이 규칙은 프로젝트 내 AI 모델 개발, 데이터 파이프라인 구축 및 데이터 전처리 작업을 담당하는 에이전트가 준수해야 하는 지침입니다.

## 1. 데이터 관리 및 전처리 규칙
- **데이터 출처 및 라이선스 기록**: 수집하거나 활용하는 모든 데이터의 출처(Source)와 라이선스(License) 정보를 명시하고 기록으로 남깁니다.
- **전처리 과정 문서화**: Raw 데이터에서 학습 데이터셋으로 정제되는 과정(결측치 처리, 이상치 정제, 피처 엔지니어링 등)을 명확하게 문서화합니다.
- **데이터 파이프라인 검증**: 결측치(Missing Value)와 이상치(Outlier) 처리를 명확히 하여 파이프라인의 안전성을 확보합니다. 데이터 로드 및 변환 시 널 값(Null) 예외 처리를 엄격히 적용합니다.

## 2. 모델링 및 실험 규칙
- **재현성(Reproducibility) 보장**: 모델 학습 및 데이터 분할(Train/Val/Test Split) 시 무작위성 제어를 위해 **난수 시드(Random Seed)를 고정**합니다.
- **정량적 평가 및 기록**: 모델의 평가지표(Metrics)를 상세히 기록합니다. 정밀도(Precision), 재현율(Recall), F1-score 등 도메인에 적합한 다각도 지표를 사용합니다.
- **다각도 실험 및 기준 모델(Baseline) 비교**: 한 가지 모델에 의존하지 않고 여러 알고리즘을 시도하며, 최종 모델은 항상 기준 모델(Baseline) 대비 개선 결과를 제시하여 타당성을 입증합니다.
- **거시적 데이터 동기화 원칙 (SSOT)**: 
  - 실시간 API 데이터와 정적 기준 데이터(좌표/등급)는 철저히 분리하여 병합해야 합니다.
  - 데모/폴백 데이터를 임의로 복제하지 않고 하나의 기준 데이터 파일(Single Source of Truth)을 공유합니다.

## 3. 시각화 및 리포팅 규칙
- **그래프 및 시각화 품질**: 모든 그래프에는 제목, 축 이름(X축, Y축), 단위 및 범례를 누락 없이 명확히 표시하여 직관성을 확보합니다.
- **결과 보고**: 실험 데이터 및 학습 성능 지표는 `docs/reports` 디렉토리 하위에 문서화하여 누적 관리합니다.
