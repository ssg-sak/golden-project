# 프론트엔드 에이전트 규칙 (FRONTEND_AGENTS.md)
Version: 1.0

이 규칙은 프로젝트의 프론트엔드 UI 화면 개발, 상태 관리 및 사용자 상호작용 설계를 담당하는 에이전트가 준수해야 하는 지침입니다.

## 1. 아키텍처 및 상태(State)와 뷰(View)의 분리
- **엄격한 레이어 분리**: 데이터 로깅, 상태 변환, 외부 API 연동을 담당하는 **컨테이너(Container/Controller) 레이어**와, 오직 props를 받아 화면에 그리기만 하는 **프레젠테이셔널(Presentational) 컴포넌트**를 분리합니다.
- **바이브코딩(Vibe Coding)을 위한 단일 책임 설계**: 컴포넌트는 단일한 역할(예: 지도 조작, 필터 UI 등)만 가지도록 제한하여 코드 복잡도 상승 및 에이전트 컨텍스트 오염을 막습니다.

## 2. 코드 품질 및 React Hooks 규칙
- **재사용성 우선 (DRY 원칙)**: 새로운 UI나 헬퍼를 만들기 전에 기존 컴포넌트 및 `shared/` 디렉토리를 먼저 검색하여 중복 코드를 최소화합니다. 구조가 유사한 UI는 복제하는 대신 `variant` props(예: `variant="admin" | "citizen"`)를 확장하여 재사용합니다.
- **React Hooks의 올바른 사용**:
  - `useRef`는 렌더링에 영향을 주지 않는 순수 DOM 참조나 인스턴스 저장용으로만 제한하여 사용합니다.
  - 상태 변화에 따른 사이드 이펙트는 `useEffect`의 의존성(Dependency) 배열을 통해 자연스럽고 제어 가능하게 다룹니다.
- **모던 개발 환경 준수 (Linter 경고 제로화)**: React 17 이상의 Automatic JSX Runtime을 활용하고, 사용하지 않는 변수나 임포트(Import) 등의 데드 코드(Dead Code)는 즉각 제거합니다.

## 3. 디바이스 및 터치 인터랙션 예외 처리
- **모바일 터치 이벤트 안티패턴 금지**: 모바일 환경에서 터치 이슈 발생 시 무지성으로 `onTouchEnd`를 덧붙여 중복 이벤트(Double-fire) 및 렌더링 버그를 양산하지 않습니다. 지도 API 또는 프레임워크의 기본 렌더링 생명주기를 먼저 파악하고 이벤트를 바인딩합니다.
- **좌표계 및 프로젝션(Projection) 연산**: 가상 좌표(Virtual Offset)나 모바일 화면 마진(Screen Offset)을 계산할 때는 타겟 줌 레벨(Target Zoom Level)의 스케일을 먼저 확정하고 논리적 연산을 적용합니다.
