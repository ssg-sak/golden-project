# 대구 골든타임

> **현재 위치에서 가까운 응급의료기관을 찾는 시민 서비스와, 대구 150개 행정동의 의료 접근성 격차를 분석하는 정책 의사결정 지원 도구를 하나의 웹서비스로 제공합니다.**

대구 골든타임은 응급상황에서 시민이 병원 위치·거리·예상 이동시간·전화·길찾기와 조회 가능한 응급실 정보를 빠르게 비교하도록 돕습니다. 동시에 골든 거버넌스 엔진은 소아·고령층 인구 수요와 의료 공급, 거리·이동시간 기반 접근성을 결합해 의료 사각지대와 자원배치 우선 검토지역을 분석합니다.

> 이 프로젝트는 공개 배포된 **개인 프로젝트·정책분석 데모**입니다. 병상 숫자나 화면의 필터 결과는 실제 환자 수용 확정을 의미하지 않습니다. 위급한 상황에서는 **119**, 응급의료 상담은 **1339**, 병원 방문 전에는 해당 의료기관의 안내를 우선하세요.


공개 주소는 GitHub Pages 배포 기록을 기준으로 연결했습니다. 저장소에는 시민 화면과 정책 화면을 대표하는 캡처 이미지가 아직 없어 깨진 이미지 링크는 넣지 않았습니다.

## 하나의 서비스, 두 가지 사용 흐름

| 구분 | 시민용 응급의료 서비스 | 골든 거버넌스 |
|---|---|---|
| 사용자 | 시민·보호자 | 정책 검토자·분석가 |
| 핵심 질문 | 지금 내 주변에서 어떤 의료기관을 확인할 수 있는가? | 어느 행정동의 접근성이 취약하고 자원 보강을 우선 검토해야 하는가? |
| 주요 입력 | 현재 위치, 병원 정적 정보, 캐시된 병상·의료정보 | 150개 행정동, 소아·고령층 인구, 의료기관, 접근성·후보지 산출물 |
| 주요 기능 | 가까운 병원 정렬, 지도·목록, 거리·ETA, 전화, 카카오 길찾기, 조건 필터 | VDI, K-Means 후보 생성, 민감도 분석, 접근성 개선 비교, 자원배치 시뮬레이션 |

## 문제 정의

### 시민 문제

가장 가까운 병원이 항상 현재 환자를 진료하거나 수용할 수 있는 것은 아닙니다. 반대로 외부 API가 실패해 병상 정보가 조회되지 않았다는 이유만으로 병원 위치·전화번호·길찾기 같은 기본 정보까지 사라져서도 안 됩니다. 시민 화면은 위치와 이동 정보를 먼저 제공하고, 조회된 병상·중증질환·의료정보의 상태를 구분해 보여주는 방식으로 이 문제를 다룹니다.

### 정책 문제

행정동별 응급의료 접근성은 의료기관 수만으로 설명하기 어렵습니다. 소아·고령층 인구 수요, 기존 병원과의 거리, 추정 이동시간, 주변 의료 인프라를 함께 봐야 합니다. 골든 거버넌스는 취약성 지수와 공간분석을 통해 후보를 만들고, 조건을 바꾼 반복 실험에서 안정적으로 남는 후보와 예상 접근성 개선 범위를 비교합니다.

## 시민용 응급의료 서비스

### 현재 위치 기반 탐색

- 브라우저 Geolocation으로 현재 위치를 요청하고 위치 상태를 별도 저장소에서 관리합니다.
- 위치가 확보되면 Haversine 거리로 병원별 직선거리를 계산해 가까운 순으로 정렬합니다.
- 지도 마커, 병원 목록, 선택 병원 상세 패널이 같은 선택 상태를 공유합니다.
- 위치 권한 거부·오류 시 안내하고, 서비스 전체가 중단되지 않도록 기본 위치 또는 목록 흐름을 유지합니다.

### 거리와 예상 이동시간

- 모든 병원은 현재 위치 기준 직선거리를 비교할 수 있습니다.
- 가까운 일부 병원은 백엔드의 Kakao Mobility 경로 API를 통해 일반 차량 ETA를 조회합니다.
- 키 누락, 호출 한도, 네트워크 실패, 일부 병원 응답 누락 시 거리 기반 추정 ETA로 전환하고 fallback 상태를 기록합니다.
- ETA는 일반 차량 경로 또는 추정치이며 119 긴급차량의 관제 경로·도착시간과 다릅니다.

### 병원 행동 기능

- 시민 목록과 사이드바의 유효한 전화번호는 tel 링크로 즉시 연결합니다.
- 카카오맵 목적지 딥링크로 일반 차량 길찾기를 새 탭에서 엽니다.
- 정책 화면의 별도 전화 아이콘에는 아직 준비 중 동작이 남아 있으므로, README에서는 시민 화면의 전화 기능만 구현 기능으로 설명합니다.

### 병상·의료정보와 필터

- NEMC mediboard와 공공데이터포털 응급의료 API 응답을 백엔드 캐시에 갱신해 응급실 병상, 특수 병상, 장비 가용 필드, 중증질환 수용 관련 필드를 병원 기본정보와 병합합니다.
- HIRA 병원 기본정보·의사 수·장비 정보를 조회하고, 장비 API가 실패하면 기준일이 표시된 스냅샷을 사용합니다.
- 진료 대상은 전체·성인·소아·고령층 흐름으로 나누고 중증질환 또는 소아 야간·휴일 조건을 추가로 필터링합니다.
- 달빛어린이병원은 응급실 병상 필드가 없는 경우가 있어 별도 소아·야간 진료 자원으로 안내합니다.
- 병상 미확인 상태와 병상 0 상태를 구분하며, 미확인을 진료 가능 또는 수용 가능으로 바꾸지 않습니다.

### 시민 안전 안내

- 화면 상단에 119·1339 안내를 제공합니다.
- 병상 숫자가 있어도 의료진, 진료과, 장비 가동, 환자 중증도와 현장 상황에 따라 실제 수용 여부는 달라질 수 있습니다.
- 카카오 길찾기는 일반 차량용입니다.
- 달빛어린이병원 운영시간과 당일 진료 여부는 방문 전 전화 확인이 필요합니다.

## 골든 거버넌스 정책분석 엔진

### 분석 범위

- 대구광역시 150개 읍·면·동의 경계와 중심점
- 0~9세 및 65세 이상 인구를 중심으로 구성한 취약 인구 수요
- 응급의료기관, 달빛어린이병원, 병원별 위치와 의료 인프라
- 행정동에서 기존 의료자원까지의 거리와 추정 이동시간
- 소아·고령층 모드별 후보지와 주변 자원 보강 필요성

### VDI와 공간 접근성

저장소의 취약성 GeoJSON과 분석 스크립트는 행정동별 취약 인구와 최근접 의료자원 접근성을 결합해 VDI 계열 지표를 계산합니다. 거리 영향과 인구 규모의 편향을 줄이기 위해 로그·정규화 필드를 함께 사용하며, 지도에서는 행정동별 취약도를 단계 색상으로 표현합니다.

이동시간 기반 분석은 전 구간의 실제 내비게이션 로그가 아닙니다. Haversine 직선거리에 거리별 우회계수, 평균속도, 기본 지연을 적용한 추정 모델이며 Kakao ETA는 시민 상세 조회와 제한된 검증 경로로 분리했습니다.

### K-Means는 후보 생성기

- 소아와 고령층의 사각지대 수요 좌표를 투영좌표계에서 군집화합니다.
- 기존 모드별 산출물은 소아 K=4, 고령층 K=3 후보 구조를 포함합니다.
- K-Means 중심을 최종 입지로 확정하지 않고 접근성 개선 검증을 위한 후보로 사용합니다.
- 수요가 적지만 지리적으로 분리된 군위 등 외곽 후보는 대구 도심 후보와 같은 의미로 단순 순위화하지 않고 별도 검토군으로 구분합니다.

### 민감도·후보지 비교

후보 민감도 분석은 K, 거리 상한, 군위 처리 조건과 난수 시드를 바꿔 후보가 반복적으로 살아남는지 검사합니다. 코드에 고정된 시드는 0, 7, 21, 42, 100이며 K-Means의 random_state와 n_init을 명시해 재현성을 높였습니다.

후보별로 다음 내용을 비교합니다.

- 후보가 개선하는 행정동과 취약 인구 규모
- 기존 최근접 자원 대비 거리·추정 이동시간 개선
- 15분·30분 접근권에 포함되는 취약 인구
- p-median·MCLP 계열 시간 기반 비교 지표
- 여러 시나리오에서 반복 등장한 비율
- 메인 대구권, 별도 지역, 보류 검토군 구분

### 의료 인프라 보강 추천

안정 후보 주변 반경 5km의 HIRA 오프라인 표본과 기존 병원 정보를 조회해 전문의, MRI, CT 보강 필요성을 1차 추정합니다. 결과는 자원배치 시뮬레이션과 정책 화면의 추천 패널로 제공하지만 실제 근무표, 당직, 운영시간, 부지·예산·법적 타당성은 포함하지 않습니다.

## 주요 결과와 정책적 시사점

- 150개 행정동을 동일한 분석 단위로 두고 소아·고령층 취약 수요와 의료 공급 접근성을 비교할 수 있는 데이터 구조를 구축했습니다.
- 원본 K-Means 결과를 그대로 확정안으로 사용하지 않고, 반복 조건에서 살아남는 안정 후보와 접근성 개선 근거를 연결했습니다.
- 소아와 고령층은 수요와 공급 기준이 달라 서로 다른 K와 후보 구조가 나타나며, 하나의 최적화 결과를 모든 계층에 공통 적용하지 않습니다.
- 외곽 지역 후보는 단순 인구 규모만으로 제거하지 않되 도심권 후보와 분리해 정책적으로 해석합니다.
- 자원 추천은 신규 병원 건립뿐 아니라 기존 병원의 전문의·MRI·CT 보강이라는 대안을 함께 검토하도록 구성했습니다.

이 결과는 후보지 현장조사, 토지·재정·법규, 실제 환자 발생·이송 기록, 소방·의료기관 협의 전 단계의 1차 분석입니다.

## API 장애와 fallback

| 상황 | 동작 | 사용자 해석 |
|---|---|---|
| 병원 API 정상 | 정적 병원정보에 캐시된 병상·의료정보 병합 | 응답·갱신 시각을 함께 확인 |
| 병상 원천 실패 | 병상 관련 필드를 null로 유지 | 미확인이며 수용 가능을 뜻하지 않음 |
| 프론트 API 실패 | 이전 정상 목록이 있으면 유지, 없으면 정적 병원 fallback 사용 | 위치·전화·길찾기 참고 가능, 병상은 최신 보장 불가 |
| HIRA 장비 실패 | 2026.03 스냅샷 또는 빈 정보 사용 | 기준일 자료 또는 미확인 |
| Kakao ETA 실패·한도 초과 | 가까운 병원의 거리 기반 추정 ETA 사용 | 실제 도로 ETA가 아닌 fallback 추정 |
| 위치 권한 실패 | 안내 후 기본 위치·목록 흐름 유지 | 사용자 실제 위치 기준이 아닐 수 있음 |
| 정책 API 실패 | 정적 분석 산출물 또는 마지막 정상 자료 사용 | 실시간 정책 데이터가 아닌 기준일 분석 |

## 실행 방법

### 프론트엔드

```bash
cd frontend
npm install
npm run dev
```

frontend/.env에 다음 값을 설정합니다.

```env
VITE_KAKAO_MAP_APP_KEY=your_kakao_javascript_key
VITE_API_BASE_URL=http://localhost:8000
```

### 백엔드

```bash
cd backend
python -m venv .venv
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

프로젝트 루트 .env에서 필요한 외부 API 키를 설정합니다.

```env
DATA_GO_KR_API_KEY=your_public_data_portal_key
HIRA_API_KEY=your_hira_key
KAKAO_REST_API_KEY=your_kakao_rest_key
SGIS_CONSUMER_KEY=your_sgis_key
SGIS_CONSUMER_SECRET=your_sgis_secret
```

키가 없거나 외부 API가 실패해도 저장소의 정적 기본정보와 명시된 fallback 경로로 제한적으로 동작합니다.

### 정책분석 파이프라인

```bash
python ai-model/run_integrated_policy_pipeline.py
python ai-model/run_integrated_policy_pipeline.py --offline
```

offline 모드는 Kakao API를 새로 호출하지 않고 기존 디스크 캐시와 분석 입력을 사용합니다.

- VDI, 공간 접근성, K-Means 후보, 민감도·후보 비교 지표 설계

## 한계와 안전상 주의사항

1. 공개 배포본은 응급의료기관·119의 공식 관제 또는 진료예약 시스템이 아닙니다.
2. 조회된 가용 병상 수는 실제 수용 확정, 대기 없음, 특정 환자 진료 가능을 의미하지 않습니다.
3. API 실패·지연·호출 한도 초과 시 이전 캐시나 정적 fallback이 표시될 수 있습니다.
4. 미확인 값은 진료 불가와 다르지만 진료 가능으로 해석해서도 안 됩니다.
5. Kakao ETA는 일반 차량 기준이며 fallback ETA는 거리 기반 추정치입니다.
6. 정책분석의 이동시간은 실제 전 구간 도로 관측값이 아닌 모델 추정치입니다.
7. VDI와 후보 분석은 사용한 인구 기준월·병원 스냅샷·거리 가정에 영향을 받습니다.
8. K-Means와 민감도 분석 결과는 우선 검토 후보이며 공식 설립지나 예산 확정안이 아닙니다.
9. 자원 추천은 반경 5km의 기준일 HIRA 표본을 사용한 1차 추정으로 운영시간·당직·부지·비용을 포함하지 않습니다.
10. 실제 정책 적용 전에는 최신 원천 재수집, 도로망·이송기록 검증, 의료기관·소방·지자체 협의가 필요합니다.
