# 대구 골든타임

현재 위치에서 가까운 응급의료기관을 찾는 **시민용 응급의료 서비스**와, 대구 150개 행정동의 의료 접근성 격차와 자원배치 대안을 분석하는 **골든 거버넌스 정책분석 엔진**을 하나의 웹서비스로 제공합니다.

시민 화면은 병원 위치·거리·예상 이동시간·전화·길찾기와 조회 가능한 응급실 정보를 제공하고, 정책 화면은 소아·고령층 수요, 의료 공급, 거리·도로 이동시간을 결합해 의료 사각지대와 우선 검토 후보를 비교합니다.

## 핵심 성과

- GitHub Pages에 시민 화면과 정책 화면을 함께 배포하고, FastAPI가 병원·병상·경로·정책 산출물을 제공합니다.
- 2026.06 인구 스냅샷을 포함한 150개 행정동 데이터를 동일 분석 단위로 구성했습니다.
- 현재 정책 산출물은 K·난수 시드·거리 상한·군위 처리 조건을 바꾼 민감도 분석으로 도시권 안정 후보, 보류 후보, 원거리 별도 검토 후보를 구분합니다. 소아·어르신 각각 240개 조건, 총 480회의 모드별 실행 결과를 검사합니다.
- **2026.07.18 검증본**은 25개 기관과 9개 후보에 대한 5,100개 도로 경로를 검증했습니다. 시설 3개 후보 조합의 30분 모델 커버리지는 소아 89.90%, 고령층 82.10%로 계산됐습니다. 이는 저장된 모델 산출물의 결과이며 실제 이송 성과나 정책 확정값이 아닙니다.

## 문제 정의

### 시민 문제

가까운 병원이 항상 특정 환자를 현재 진료하거나 수용할 수 있는 것은 아닙니다. 반대로 외부 API 장애로 병상 정보가 조회되지 않았다는 이유만으로 병원 위치·전화·길찾기 같은 기본 정보까지 사라져서도 안 됩니다.

### 정책 문제

의료 접근성은 의료기관 수만으로 설명하기 어렵습니다. 취약 인구 규모, 최근접 의료자원까지의 거리와 도로 이동시간, 소아·고령층별 수요·공급 조건을 함께 보아야 합니다.

## 시민용 응급의료 서비스

### 현재 위치 기반 탐색

- 브라우저 Geolocation으로 현재 위치를 요청합니다.
- 위치를 확보하면 Haversine 직선거리를 계산해 가까운 병원부터 정렬합니다.
- 지도 마커, 병원 목록, 선택 병원 상세가 같은 선택 상태를 공유합니다.
- 위치 권한 거부나 오류가 발생해도 기본 위치·목록 흐름을 유지합니다.
- 데스크톱 사이드바와 모바일 하단 시트로 동일한 탐색 흐름을 제공합니다.

### 거리·예상 이동시간·행동 기능

- 모든 병원에 현재 위치 기준 직선거리를 표시합니다.
- 가까운 최대 5개 병원은 백엔드의 Kakao Mobility 경로 API로 일반 차량 ETA를 조회합니다.
- API 키 누락, 호출 한도, 네트워크 오류, 일부 응답 누락 시 직선거리 × 1.35, 평균속도 38km/h, 기본 지연 3분을 사용한 추정 ETA로 전환합니다.
- 유효한 전화번호는 전화 링크로 연결하고, 카카오맵 목적지 딥링크로 길찾기를 엽니다.
- ETA는 일반 차량 경로 또는 추정치이며 119 긴급차량의 관제 경로·도착시간과 다릅니다.

### 병상·의료정보·필터

- NEMC mediboard와 공공데이터포털 응급의료 응답을 백엔드 캐시에 수집해 병원 기본정보와 병합합니다.
- 병상 캐시 폴링은 기본 120초이며 설정값은 60~300초 범위로 제한됩니다.
- 전체·성인·소아·고령층 흐름과 중증질환 조건을 분리해 필터링합니다.
- 달빛어린이병원은 응급실 병상 필드가 없는 소아 야간·휴일 진료 자원일 수 있어 별도로 안내합니다.
- 병상 미확인(null)과 병상 0을 구분하며 미확인을 진료 가능 또는 수용 가능으로 바꾸지 않습니다.

### 시민 안전 안내

- 화면에서 119와 1339를 안내합니다.
- 병상 수가 표시돼도 의료진, 진료과, 장비 가동, 환자 중증도와 현장 상황에 따라 실제 수용 여부는 달라질 수 있습니다.
- 달빛어린이병원 운영시간과 당일 진료 여부는 방문 전 전화 확인이 필요합니다.

## 골든 거버넌스 정책분석 엔진

### 핵심 분석 질문

1. 취약 인구가 많으면서 기존 응급의료 자원과 먼 행정동은 어디인가?
2. 소아와 고령층의 취약지역·후보 구조는 어떻게 다른가?
3. 후보를 추가할 때 평균 이동시간과 15분·30분 접근권은 얼마나 달라지는가?
4. 분석 조건을 바꿔도 반복해서 나타나는 후보는 어디인가?

### 분석 범위

- 대구광역시 150개 읍·면·동 경계와 중심점
- 0~9세와 65세 이상 인구
- 응급의료기관·달빛어린이병원 위치
- 행정동에서 기존 의료자원·후보지까지의 직선거리, 추정 이동시간, Kakao 도로 경로 시간
- 소아·고령층 모드별 후보, 민감도, 도로 접근성·입지 최적화 산출물

### 방법론 요약

| 방법 | 왜 사용했는가 | 변수·계산 | 해석 | 한계 |
|---|---|---|---|---|
| VDI | 인구 수요와 의료 접근성 불리함을 한 지표로 비교 | 취약인구 = 0~9세 + 65세 이상. 후보 생성 전 단계는 log(1+최근접 거리) × 취약인구, 현재 공개 분석본은 log(1+실제 도로 ETA) × 취약인구를 사용 | 값이 클수록 인구 규모와 이동시간 조건을 함께 검토할 필요가 큼 | 행정동 중심점·경로 조회 시점·기준일에 민감하며 정책 판단을 대체하지 않음 |
| K-Means | 의료 사각지대 점의 공간적 중심을 후보로 생성 | EPSG:5179 투영좌표의 사각지대 점, K=2~5, 시나리오별 고정 seed와 n_init=10 | 군집 중심은 후속 접근성 검증을 위한 후보 | 현재 후보 생성은 개별 점을 동일 가중치로 취급하며 비구형 수요·외곽지역·K 선택에 민감함 |
| 민감도 분석 | 특정 K나 시드에만 의존한 후보를 걸러냄 | K=2~5, seed=0·7·21·42·100, 거리 상한 없음·10·15·20km, 군위 포함·분리·제외 | 반복 등장 비율로 안정 후보·보류·별도 권역을 구분 | 시나리오 범위 밖 정책 조건은 평가하지 않음 |
| 도로 접근성 | 직선거리 후보를 실제 도로 경로 시간으로 재검증 | Kakao Mobility Directions API와 디스크 캐시, 행정동-병원·후보 ETA 행렬 | 기존 자원 대비 이동시간 개선과 접근권 변화를 비교 | 교통 시점·경로 API 오차가 있고 119 관제 경로가 아님 |
| p-median·MCLP | 평균 시간과 시간권 커버리지를 서로 다른 목표로 비교 | 시설 1~3개 조합 전수 비교, 인구가중 평균 ETA 최소화, 15분·30분 커버 인구 최대화 | 목표에 따라 최적 조합이 달라짐을 보여줌 | 부지·예산·인력·법적 조건과 실제 환자 흐름은 미반영 |

### 분석 이미지

[안정 정책 후보 시각화](./data/processed/stable_policy_candidates_overview_20260715.png)는 소아·고령층 민감도 분석에서 반복 등장한 9개 검토 후보를 보여줍니다.

새 서비스 화면 캡처는 추가하지 않았습니다. 시민 화면과 정책 화면은 상단의 배포 서비스에서 직접 확인할 수 있습니다.

### 데이터 제공 방식 구분

- **실시간·준실시간:** 외부 API를 조회해 백엔드 캐시에 보관한 병상·의료 필드입니다. 원천 갱신과 캐시 주기만큼 지연될 수 있습니다.
- **정적:** 병원 위치·주소·전화·분류처럼 저장소 정본에서 제공하는 기본정보입니다.
- **기준일 분석:** 2026.06 인구와 2026.07.18 정책 분석본처럼 기준 시점이 고정된 자료입니다.
- **fallback:** API 장애 때 이전 정상 목록 또는 정적 병원 정본을 사용합니다. 병상은 null로 유지해 허위 수용 가능 표시를 막습니다.
- **mock·test:** 테스트 전용이며 실제 조회 데이터처럼 설명하지 않습니다.

병상 응답은 캐시 생성 시각과 stale 상태를 함께 전달합니다. 프론트는 API를 조회한 시각을 병상 갱신 시각으로 대신 표시하지 않으며, 캐시가 오래됐거나 마지막 갱신이 실패하면 `마지막 정상 자료`임을 안내합니다.

  A --> C[행정동·인구 결합과 품질 검증]

  G --> H[p-median·MCLP 조합 비교]

~~~

현재 정책 화면은 **2026.07.18 검증본** 하나만 사용합니다. 내부 데이터 계보 식별자는 `2026-07-18-r2`로 유지하지만 사용자 화면에는 노출하지 않습니다. 정책보고서는 25개 기관·후보 9곳·도로 경로 5,100건과 현재 코드 구성을 모두 대조한 뒤 고정 스냅샷으로 연결하며, 검증 전 파일은 현재 분석 근거로 공개하지 않습니다.

이번 데이터 무결성·갱신 안전성 개선의 결함 원인, 수정 범위, 검증 결과와 외부 운영 조건은 [데이터 무결성·갱신 안전성 강화 보고서](./docs/DATA_INTEGRITY_AND_REFRESH_HARDENING_20260718.md)에 기록했습니다. ChatGPT 독립심사 78점 판정의 지적사항별 원인·조치·증빙·잔여 위험은 [독립심사 후속 개선 보고서](./docs/CHATGPT_REVIEW_REMEDIATION_20260718.md)에서 별도로 확인할 수 있습니다.

정책 분석본의 고위험 경계값은 의료적 절대 임계값이 아닙니다. 150개 행정동의 VDI를 높은 순서로 정렬한 뒤 **상위 25%를 우선 확인 대상으로 구분하는 현재 분석본 내부의 상대 경계값**이며, 인구·기관·도로 입력이 바뀌면 함께 달라집니다.

### 정책자료를 읽는 방법

- 정책 화면의 위험도·후보·접근성 비교는 현재 저장된 기준자료를 바탕으로 한 분석본입니다.
- 정책 화면은 기관·행정동·후보·도로 경로·최적화 결과가 같은 분석 기준으로 검증된 경우에만 공개합니다.
- 인구·병원·행정동 원천이 바뀌면 150개 행정동과 25개 기준 의료기관 구조를 먼저 확인한 뒤 VDI와 정책 산출물을 다시 계산합니다.
- 새 분석은 5,100개 도로 경로와 입력 해시 검증을 모두 통과한 뒤에만 공개 파일로 승격하며, 실패하면 직전 공개본을 유지합니다.
- 후보 정본 9곳과 후보 추적 9곳의 모드·ID·좌표가 일치해야 하며, 후보 추적 해시도 단일 정책 분석본에 기록합니다.
- 후보지와 p-median·MCLP 조합 결과는 데이터·거리·목적함수 조건에 따라 달라질 수 있으며, 실제 정책 확정이나 의료적 판단을 뜻하지 않습니다.

### 자동 갱신의 안전 경계

- `ENABLE_PUBLIC_DATA_SCHEDULER=false`가 기본값이며, 운영 검증 전에는 자동 수집을 켜지 않습니다.
- 병원·달빛어린이병원은 일 단위, 전체 인구 확인은 월 단위, 행정동 확인은 월 단위 작업으로 분리합니다.
- 스케줄러는 원천 수집과 변경 감지만 수행하며 정책 공개본을 자동 승격하지 않습니다.
- 각 예약 작업은 시작·성공·실패·소요시간·변경 여부를 로그로 남기고, 단일 인스턴스에서는 DB 작업 잠금과 APScheduler의 `max_instances=1`로 중복 실행을 막습니다.
- 현재 인구 API 스냅샷은 전체 인구만 저장하므로 0~9세·65세 이상 인구로 비율 추정하지 않습니다. 공식 연령별 150개 행정동 입력과 기준월이 일치하지 않으면 재분석을 중단하고 기존 검증본을 유지합니다.
- 정책 릴리스에는 인구 CSV·기준월 manifest·민감도 원본의 SHA-256과 모드별 완료 건수를 기록합니다. 기준월 문자열 또는 일부 산출물만 임의로 바꾼 분석본은 공개 계약을 통과할 수 없습니다.
- SGIS API 갱신은 행정동 목록·중심점 변경 감시 단계입니다. 정책 공간분석은 저장된 검증 경계 GeoJSON을 사용하며, 경계 변경을 자동 공개본으로 승격하지 않습니다.
- SQLite 잠금은 단일 Render 인스턴스 범위의 보호입니다. 다중 인스턴스 운영으로 확장할 때는 외부 분산 락이 추가로 필요합니다.

## 실행 방법

### 프론트엔드

~~~bash
cd frontend
npm install
npm run dev
~~~

필요한 프론트 환경변수는 VITE_KAKAO_MAP_APP_KEY와 VITE_API_BASE_URL입니다.

### 백엔드

~~~bash
cd backend
python -m venv .venv
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
~~~

외부 연동에는 DATA_GO_KR_API_KEY, KAKAO_REST_API_KEY, SGIS_CONSUMER_KEY, SGIS_CONSUMER_SECRET 등이 필요합니다. 실제 값은 저장소 문서에 기록하지 않습니다.

### 정책분석 파이프라인

~~~bash
python ai-model/run_integrated_policy_pipeline.py
python ai-model/run_integrated_policy_pipeline.py --offline
~~~

offline 모드는 Kakao API를 새로 호출하지 않고 기존 디스크 캐시와 분석 입력을 사용합니다.

## 한계와 안전상 주의사항

1. 외부 API 장애·호출 한도·갱신 지연으로 이전 캐시나 정적 fallback이 표시될 수 있습니다.
2. 조회 실패는 실제 병상 부족과 다르지만 진료 가능·수용 가능으로 해석해서도 안 됩니다.
3. 병상 수가 있어도 실제 의료진·진료과·장비·환자 상태에 따라 수용 여부가 달라질 수 있어 전화 확인이 필요합니다.
4. Kakao ETA는 일반 차량 기준이고 fallback ETA는 거리 기반 추정치입니다.
5. 인구·병원·정책 산출물의 기준 시점이 서로 다릅니다.
6. VDI 공식과 보조 가중치는 분석 목적의 선택이며 다른 가중치에서 순위가 달라질 수 있습니다.
7. K-Means는 후보 생성기이며 비구형 공간, 외곽 수요, K·시드 설정에 민감합니다.
8. 의료기관의 실제 수용능력, 의료진 근무, 장비 가동, 환자 발생·이송 기록이 충분히 반영되지 않았습니다.
9. 토지·예산·인력·법적 제약과 외부 의료·소방·지자체 전문가 검증은 포함하지 않았습니다.
10. 이 서비스와 분석은 의료적 판단, 응급 신고, 공식 정책 결정을 대체하지 않습니다.
