# 대구 골든타임: Golden Governance

대구광역시 행정동별 응급의료 접근성과 취약인구를 함께 분석해, 현장 조사 전에 먼저 살펴볼 정책 검토 후보를 좁히는 데이터 분석·대시보드 프로젝트입니다.

## 핵심 정리

| 구분 | 내용 |
| --- | --- |
| 분석 지역 | 대구광역시 |
| 분석 단위 | 행정동, 유치원·취약 수요 지점, 병원 주변 3km/5km 탐색 범위 |
| 주요 대상 | 소아 야간·휴일 진료 수요, 고령층 중증 응급 접근성 수요 |
| 핵심 방법 | GIS 공간 필터링, VDI, 인프라 패널티, K-Means 기반 후보 생성, 민감도 분석 |
| 결과물 | 정책 우선 검토 후보 JSON, 자원 보강 시나리오, React 정책 대시보드 |
| 현재 상태 | 민감도 분석에서 반복 등장한 후보만 정책탭에 표시하고, 모델 결과는 PoC 참고 자료로 동결 |

현재 결과는 확정 입지나 정책 처방이 아닙니다. K-Means는 실제 입지 최적화 모델이 아니라 수요 중심 후보를 찾는 후보 생성기로 사용했습니다. 실제 정책 결정에는 현장 조사, 교통 시간, 부지 가능성, 병원별 역할, 예산, 의료·행정 전문가 검토가 추가로 필요합니다.

## 모델 해석 원칙

분석 결과가 정책 확정안처럼 읽히지 않도록, 프로젝트 설명과 화면 문구는 다음 원칙으로 정리했습니다.

- `최적 입지`, `AI 추천`, `거점 확정`처럼 강한 표현을 `정책 우선 검토 후보`, `검증용 산출물`, `참고 자료`로 낮춤
- 병원 전문의 수, MRI/CT, 일부 위치 데이터가 오프라인 덤프·추정치 기반 PoC 데이터임을 명시
- 3km/5km 기준은 정답이 아니라 시나리오 기준이며 민감도 분석과 추가 검증이 필요하다고 표시
- K-Means는 p-median, MCLP, location-allocation 같은 실제 입지 최적화 모델과 비교가 필요한 후보 생성기라고 명시
- 소아 야간·휴일 진료와 고령층 중증 응급 접근성은 같은 정책 문제로 합치지 않고 분리해서 해석

## 프로젝트를 시작한 이유

현재 거주 중인 달성군 옥포읍은 응급의료 인프라가 상대적으로 취약한 지역입니다. 가족에게 응급 상황이 발생했을 때 “가까운 병원이 있다는 사실”만으로는 충분히 안심할 수 없다는 문제의식에서 출발했습니다. 병원까지의 거리뿐 아니라 취약인구 규모, 전문의·장비 등 인프라 상태를 함께 보아야 실제 정책 논의에 가까워질 수 있다고 판단했습니다.

## 핵심 질문

- 대구에서 소아와 고령층의 응급의료 접근성 취약지는 각각 어디인가?
- 병원과의 거리뿐 아니라 병원 인프라 상태를 함께 보면 우선 검토 지역이 어떻게 달라지는가?
- 신규 시설 설치, 기존 병원 기능 보강, 이송·교통 지원 중 어떤 정책 논의를 먼저 시작할 수 있는가?
- 모델이 낸 좌표를 정답처럼 쓰지 않으려면 어떤 한계와 검증 조건을 함께 제시해야 하는가?

## 주요 기능

| 기능 | 설명 | 상태 |
| --- | --- | --- |
| 계층별 취약지대 탐색 | 소아와 고령층을 분리해 병원 접근성 취약 수요 지점을 필터링 | 구현 완료 |
| VDI 기반 정책지표 | 거리 부담과 취약인구를 함께 보는 참고 지표 제공 | 구현 완료 |
| K-Means 후보 생성 | 복합 가중치를 `sample_weight`로 적용해 수요 중심 후보 생성 | 구현 완료 |
| 후보 민감도 분석 | K, seed, 거리 상한, 군위 처리 조건을 바꿔 반복 등장 후보 확인 | 구현 완료 |
| 자원 보강 시나리오 | 후보 주변 5km 병원 인프라를 바탕으로 보강 필요성을 1차 추정 | 구현 완료 |
| 정책 대시보드 | 취약지역, 검토 후보, 지표 해설, 데이터 한계를 Kakao Maps 기반 화면에 표시 | 구현 완료 |
| 교통망 기반 이동시간 | 직선거리 대신 실제 이동시간을 반영하는 분석 | 향후 개발 |

## 분석 흐름

```mermaid
flowchart LR
    A[공공데이터 수집] --> B[정제와 좌표계 통일]
    B --> C[소아 / 고령층 수요 분리]
    C --> D[3km 사각지대 후보 필터링]
    D --> E[VDI와 인프라 패널티 계산]
    E --> F[K-Means 후보 생성]
    F --> G[민감도 분석과 안정 후보 분리]
    G --> H[5km 자원 보강 시나리오]
    H --> I[정책 대시보드 표시]
```

## 데이터 상태와 해석 제한

| 데이터 | 상태 | 해석 |
| --- | --- | --- |
| 행정동 경계·인구 | 공공데이터 기반 | 본 분석에 사용 가능 |
| 병원 위치 | 공공데이터 기반, 일부 정제 필요 | 지도 표시와 거리 계산에 사용 |
| 병원 전문의 수 | HIRA 오프라인 덤프·추정치 포함 | 정책 확정 근거가 아니라 구조 시연용 |
| MRI/CT 보유 정보 | 오프라인 덤프·추정치 포함 | 보강 필요성 탐색용 참고값 |
| 3km/5km 거리 기준 | 분석자가 설정한 시나리오 | 실제 생활권·이송시간과 다를 수 있음 |
| K-Means 후보 좌표 | 수요 중심 후보 생성 결과 | 최종 입지 결정값이 아님 |

## 실행 방법

### 백엔드

```bash
cd backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 프론트엔드

```bash
cd frontend
npm install
npm run dev
```

필요한 환경변수 예시는 다음과 같습니다.

```env
VITE_KAKAO_MAP_API_KEY=your_kakao_api_key
VITE_API_BASE_URL=http://localhost:8000
```

### 분석 파이프라인

```bash
cd ai-model
pip install -r ../backend/requirements.txt
python golden_governance_pipeline.py
python resource_simulator.py
```

## 한계

- 작성자는 의료·보건·행정 전문가가 아닌 부트캠프 교육생이며, 이 프로젝트는 학습과 포트폴리오 목적의 PoC입니다.
- 직선거리와 공간 버퍼는 실제 이송 시간, 도로망, 교통 체증, 생활권을 완전히 반영하지 못합니다.
- 병원별 전문의 수와 장비 정보는 최신 공식 운영 데이터와 다를 수 있습니다.
- 자원 보강 수치는 예산 산정이나 정책 처방이 아니라, 어떤 정보를 추가로 확인해야 하는지 보여주는 탐색용 시나리오입니다.
- 실제 입지 최적화에는 p-median, maximal covering location problem, location-allocation 모델과의 비교가 필요합니다.

## 향후 개선

- 실제 이동시간 기반 접근성 분석 추가
- 병원별 최신 인프라 데이터 확보와 검증
- p-median, MCLP 등 입지 최적화 모델과 K-Means 후보 비교
- 의료·행정 전문가 검토를 반영한 가중치 재설계
- 후보별 1페이지 설명 카드와 지도 기반 검증 리포트 추가
