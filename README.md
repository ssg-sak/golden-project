# 대구 골든타임 (Daegu Golden Time)

> **"위급 상황에서 1초라도 빨리 진료 가능한 병원을 찾고, 데이터로 응급 의료 사각지대를 관제합니다."**

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT) 
[![FastAPI](https://img.shields.io/badge/FastAPI-005571?style=flat&logo=fastapi)](https://fastapi.tiangolo.com)
[![React](https://img.shields.io/badge/React-20232A?style=flat&logo=react&logoColor=61DAFB)](https://reactjs.org/)

**대구 골든타임**은 단순한 병원 지도 앱이 아닙니다. 이 프로젝트는 **'응급실 뺑뺑이'**를 방지하기 위해 시민에게 가장 빠른 이동 경로를 안내하고, 동시에 행정·정책 관리자에게는 지역 응급 인프라의 붕괴 위험을 시각적으로 관제하게 해주는 **투트랙(Two-Track) 응급의료 거버넌스 플랫폼**입니다.

## 🎯 핵심 타겟과 듀얼 뷰 (Dual-View) 아키텍처

이 플랫폼은 **하나의 웹 앱 안에서 두 명의 완전히 다른 타겟을 위한 뷰(View)를 제공**합니다. 상단의 탭(Tab) 전환을 통해 모드가 완전히 분리됩니다.

### 🚑 탭 A. 시민 (응급 상황) 모드
- **목적:** "1분 1초가 급한 상황에서, 당장 나를 받아줄 수 있는 가장 빨리 도착할 병원은 어디인가?"
- **주요 기능:**
  - **카카오내비 ETA (Estimated Time of Arrival) 실시간 연동:** 직선 거리가 아닌 현재 교통 상황이 반영된 **'도착 예상 시간'**을 기준으로 병원 목록을 실시간 자동 정렬합니다. (예: `🚗 12분 소요`)
  - **가용 병상 시각적 필터링:** 응급실이 꽉 차서 수용 불가(0)인 병원은 붉게 표시되거나 하단으로 밀려나 구급대원과 시민의 헛걸음을 방지합니다.
  - **미니멀 UX:** 거추장스러운 복잡한 인프라 정보는 숨기고, 오직 '이동 시간'과 '진료 가능 여부'에만 집중합니다.

### 📊 탭 B. 정책 관리자 (관제) 모드
- **목적:** "대구 지역의 권역별 응급 인프라는 탄탄한가? 의료 공백(사각지대)은 어디인가?"
- **주요 기능:**
  - **의료 인프라(전문의/장비) 팩트 관제:** 심평원(HIRA) 데이터를 바탕으로 각 병원의 CT/MRI 보유 현황과 전문의 수를 팝업 패널에 즉각 표출합니다.
  - **AI 기반 사각지대 지수(VDI) 분석:** 고립된 동네(수요는 높으나 병원이 먼 곳)를 AI(K=4 파인튜닝) 공간 분석을 통해 도출하고 지도에 붉은 히트맵으로 시각화합니다.

## 🏗️ 화면 구조 (3-Panel Layout)

웹사이트는 한 화면에서 시선의 분산을 막고 정보의 밀도를 높이기 위해 **3-Panel 아키텍처**를 사용합니다.

1. **좌측 사이드바 (List & Navigator):** 시민/관리자 탭을 전환하며, 탭 성격에 맞춰 'ETA 기준 가까운 순' 또는 '등급(Tier) 순'으로 병원 리스트를 나열합니다.
2. **중앙 메인 캔버스 (Interactive Map):** 카카오맵 API 기반. 마커 색상(병상 유무)과 히트맵(VDI)이 실시간으로 교차 렌더링됩니다.
3. **우측 상세 패널 (Detail Information):** 마커 클릭 시 나타나는 슬라이드 패널로, 시민 모드에서는 [카카오내비 즉시 연결], 정책 모드에서는 [의료 인프라 현황]이 표출됩니다.

### 🧩 프론트엔드 컴포넌트 아키텍처 (FSD 기반)

본 프로젝트는 상태와 뷰의 엄격한 분리를 위해 일반적인 단일 `components` 폴더가 아닌 **Feature-Sliced Design(FSD)** 레이어 분리를 차용하였습니다.
- **`app/`**: 진입점 및 전역 설정
- **`widgets/`**: 독립적 비즈니스 로직을 포함하는 UI 블록 (사이드바, 맵 캔버스 등)
- **`shared/`**: 상태를 가지지 않는 순수 프레젠테이셔널 UI 및 공통 유틸리티
- **`data/`**: API 연동 및 상태 관리 (Container 역할)

자세한 구조는 [코드 학습·설명서 (CODE_EXPLANATION.md)](./CODE_EXPLANATION.md)를 참조하세요.

## 🛡️ 데이터 철학과 안정성 (System Reliability)

공공의료 시스템에서 **'보기에 예쁜 가짜 데이터(Mock)'는 절대 용납될 수 없는 시스템 조작**입니다. 본 프로젝트는 데이터 무결성과 시스템 무중단(Zero-Downtime)을 위해 강력한 방어 기제를 탑재했습니다.

- **가짜 데이터(Mock)의 철저한 배제:** 모든 데이터는 국립중앙의료원과 심평원(HIRA)의 팩트에 기반합니다.
- **🧪 시뮬레이션(스냅샷) 데모 모드:** 포트폴리오 심사를 위해 임의의 조작된 가짜 데이터를 하드코딩하여 시스템 무결성을 훼손하는 것을 엄격히 금지합니다. 대신, 과거 혼잡도가 극에 달했던 특정 시점의 **'실제 공공데이터 캡처본(Snapshot)'**을 내장하여, 백엔드 없이도 화려한 UI/UX(수용 불가 병원 마커 변화, 장비 고장 응급 메시지 등)를 데이터 윤리 훼손 없이 시연할 수 있는 데모 토글을 제공합니다.
- **오프라인 덤프 폴백 (Offline Fallback):** 심평원 API가 403, 500 에러를 반환하며 서버가 다운되더라도 화면이 터지거나 공란이 나오지 않습니다. 대구 지역 24개 응급의료기관의 규모별 팩트(조사 추정치)를 백엔드에 **하드코딩(오프라인 덤프)**하여, 서버 부하(N+1 쿼리) 제로 상태에서 1밀리초 만에 화면에 전투력 지표(전문의, CT/MRI)를 표출합니다.
- **3초 서킷 브레이커:** 외부 공공 API 응답이 3초를 초과하면 자동으로 연결을 끊고 즉시 로컬 캐시(또는 정적 null 폴백) 데이터를 띄워 '우아한 성능 저하(Graceful Degradation)'를 달성합니다.

## 🚀 빠른 시작 (Quick Start)

본 프로젝트는 Node.js 20+ 및 Python 3.11+ 환경을 권장합니다.

### 1. 환경 변수 설정
`frontend/.env` 및 프로젝트 루트 `.env` 파일에 발급받은 API 키를 기입하세요.
- `VITE_KAKAO_MAP_APP_KEY`: 카카오맵 JavaScript API 키
- `DATA_GO_KR_API_KEY`: 국립중앙의료원 API 키

### 2. 서버 실행

```bash
# [터미널 1] 백엔드 (FastAPI) 실행
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# [터미널 2] 프론트엔드 (Vite/React) 실행
cd frontend
npm install
npm run dev
```

브라우저에서 `http://localhost:5173/` 로 접속하면 듀얼 뷰 대시보드가 구동됩니다.
