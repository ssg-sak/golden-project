## 1. 문제 분석

### 피드백 요약

- 목록에만 보이던 병원들을 **한눈에 지도에** 보고 싶다.
- *(비유)* 네이버 지도 검색처럼 **마커가 한꺼번에** 뜨는 느낌 → **구현은 카카오맵**.
- **직선 거리만**으로는 실제 이동이 불편할 수 있다.
- 지도와 목록을 **같이** 보면서 고르고 싶다.
- 병원마다 따로 들어가야 지도를 볼 수 있어 불편하다.
- *(후속)* 상세에 들어가면 **지도와 병원 정보가 같이** 보이면 좋겠다.
- *(후속)* 상세에서 **스크롤이 안 내려가는** 버그.

### 갭

| 구분 | 이전 (모바일) | 현재 목표 |
|------|---------------|-----------|
| 탐색 UI | 목록 전용 | 목록 + 지도 동시 |
| 지도 | 데스크톱(`lg+`)만 | 모바일 임베드 |
| 비교 | 거리·ETA 숫자 | 위치·방향(직선) 감각까지 |
| 상세 | 목록 → 정보만 전체 화면 | **상단 지도 + 하단 상세** |
| 스크롤 | 높이 체인 깨짐으로 멈춤 | 고정 셸 + 패널 내부 스크롤 |

핵심: “지도가 없다”가 아니라 **후보 위치를 한 화면에서 비교할 수 없다** → 선택 후에도 **위치가 사라지면 안 된다**.

## 2. 목표 · 비목표

**목표**

1. 필터된 병원 마커를 모바일 카카오맵에 일괄 표시
2. 목록과 지도를 같은 탐색 화면에서 연동
3. 거리·병상·ETA 유지 + 지도로 위치·경로 감각 보완
4. 상세 진입 시에도 **지도 + 병원 정보**를 함께 표시
5. 상세 본문이 **안정적으로 스크롤**되고, 물리 Back·불투명 헤더 UX 유지

**비목표**

- 네이버 지도 SDK/API 도입
- Directions SDK·도로 폴리라인 API 전면 도입 (직선 Polyline만)
- 관리자 모바일 지도 개편

## 3. UX · 동작 규칙 (현재)

### 탐색 화면

```text
┌─────────────────────────┐
│ 필터 (진료대상·병상 등)   │
├─────────────────────────┤
│ 보기: 균형 / 지도 크게…  │  ← 프리셋
├─────────────────────────┤
│     카카오맵 + 마커       │  ← 높이: 프리셋 또는 드래그
├───── 드래그 핸들 ────────┤
│ 목록 (거리·병상·ETA)     │
└─────────────────────────┘
```

### 상세 화면 (지도 사용 가능 시)

```text
┌─────────────────────────┐
│ ← 병원 목록으로          │  ← GNB 아래 fixed 셸
├─────────────────────────┤
│  카카오맵 (선택 병원 강조 │
│  + 내 위치↔병원 직선)    │
├─────────────────────────┤
│  병원 상세 본문          │  ← layout="panel"
│  (overflow-y-auto만)     │     내부 스크롤
└─────────────────────────┘
```

| 입력 | 결과 |
|------|------|
| 목록 카드 탭 | `selectedHospital` → 상세+지도 화면 |
| 지도 마커 탭 | 동일하게 상세+지도 (다른 마커면 병원 전환, history 1단 유지) |
| 「병원 목록으로」 / 물리 Back | `closeDetail` → 탐색 화면 복귀 |
| 보기 프리셋 / 드래그 | 탐색 화면에서만 지도 높이 조절 |
| 카카오 실패 | 탐색은 목록만 · 상세는 **정보만** fixed 전체 화면 |

## 4. 구현 결과 (단계별)

### 1단계 — 지도+목록 임베드

| 항목 | 내용 |
|------|------|
| 레이아웃 | 상단 카카오맵 + 하단 목록 |
| 마커 | 필터된 병원 일괄 표시 |
| 폴백 | 키 미설정·로딩·오류 시 목록만 |
| Back | `useMobileHospitalDetailHistory` 유지 |
| 데스크톱 | `lg` 분기 — 지도 단일 마운트 |

**현장 이슈:** `basis` 비율만으로는 맵 `height:100%` ≈ 0 → 빈 패널.  
**조치:** 명시적 픽셀/`dvh` 높이 + `absolute inset-0` + `onCreate` 시 `relayout()`.

### 2단계 — 보기 비율 프리셋

- 「균형 / 지도 크게 / 목록 크게」로 지도 높이 전환

### 3단계 — 드래그 + 직선 경로

| 항목 | 내용 |
|------|------|
| 드래그 | 지도·목록 사이 핸들 (min 120px ~ 뷰포트 65%) |
| 프리셋 | 클릭 시 드래그 높이를 프리셋으로 리셋 |
| 직선 | 내 위치↔선택(또는 포커스) 병원 `Polyline` (도로 경로 아님) |

초기에는 마커=미리보기 칩 / 목록=상세로 분리했다가,  
피드백 “상세에 지도도 같이”에 맞춰 **마커·목록 모두 상세+지도**로 단순화했다.

### 4단계 — 상세+지도 · 스크롤 복구

| 항목 | 내용 |
|------|------|
| 상세 UI | 상단 지도(~38vh, clamp) + 하단 `HospitalDetailPanel` |
| 셸 | `fixed` + `top: var(--mobile-nav-height)` + `bottom: 0` — **뷰포트 높이 확정** |
| 스크롤 | 패널 `layout="panel"` → 본문만 `overflow-y-auto` + `touch-pan-y` |
| 실패 폴백 | 지도 없으면 상세만 같은 fixed 셸 |

**스크롤 회귀 원인:** flex `flex-1`만으로는 조상 높이 체인이 깨져 `overflow-y-auto`가 스크롤 박스를 못 만듦.  
**조치:** 상세를 문서 흐름이 아닌 **고정 뷰포트 셸**에 올리고, 남은 영역에 `min-h-0 flex-1` + 패널 내부 스크롤.

커밋 참고: `3aff5c8` `fix: restore mobile hospital detail scroll with map`
