이 문서는 현재 저장소와 연결된 개인 포트폴리오를 다음 2개 축으로 정리하기 위한 실행 로드맵이다.

## 3. 포지셔닝

### 대구 골든타임

대구 골든타임은 전체 포트폴리오의 중심 프로젝트다. 시민은 가까운 응급의료기관과 최근 조회 병상정보를 확인하고, 정책 관점에서는 150개 행정동, 25개 응급 관련 기관, 9개 후보지, 5,100개 도로 경로를 바탕으로 접근성 격차를 확인한다.

이 프로젝트는 더 이상 무한 확장 대상으로 보지 않는다. 현재 검증된 릴리스를 기준으로 동결하고, 남은 작업은 설명력과 증거 정리에 집중한다.

### 골든 데이터랩

골든 데이터랩은 대구 골든타임의 검증 정책 스냅샷을 별도 분석가 포트폴리오로 확장하는 프로젝트다. 서비스 운영 코드를 건드리지 않고, 고정된 정책 릴리스 데이터를 SQL, Python, BI 관점에서 다시 분석한다.

데이터랩에서는 실시간성을 주장하지 않는다. 기준 스냅샷을 분석하고, 향후 월별 스냅샷이 추가되면 버전 간 변화 비교로 확장한다.

## 7. 데이터 갱신 표현 원칙

골든 데이터랩과 포트폴리오 설명에서는 다음 표현을 사용한다.

| 표현 | 사용 여부 | 이유 |
|---|---|---|
| 실시간 정책분석 | 사용하지 않음 | 현재 검증 대상은 고정 정책 릴리스이기 때문 |
| 월별 자동 갱신 완료 | 사용하지 않음 | 아직 운영 증거가 부족함 |
| 검증 스냅샷 기반 분석 | 사용 | 현재 구조와 증거에 맞음 |
| 향후 월별 스냅샷 비교 가능 | 사용 | 확장 방향으로 설명 가능 |
| 최근 조회 병상정보 | 사용 | 시민 화면의 병상 표현 원칙과 일치 |

## 8. 최종 포트폴리오 메시지

최종 포트폴리오에서는 다음 흐름으로 설명한다.

1. 대구 골든타임으로 공공데이터 기반 시민 서비스와 정책분석 엔진을 구현했다.
2. 25개 기관 기준으로 전면 재분석하고, 데이터 계약과 CI로 검증 가능한 기준 릴리스를 만들었다.
3. 골든 데이터랩으로 같은 검증 스냅샷을 SQL, Python, BI 분석 포트폴리오로 확장했다.
4. 서비스 구현 역량과 데이터 분석 역량을 하나의 문제의식 아래 연결했다.

## 9. 성공 기준

3주 후 성공 기준은 다음과 같다.

- 대구 골든타임 README, 보고서, 배포 링크, 검증 문서가 서로 모순되지 않는다.

- 대구 골든타임 README에서 배포 서비스와 핵심 문서로 바로 이동할 수 있다.
- 골든 데이터랩에 Python EDA, SQL 분석, BI 설계 또는 대시보드 초안이 존재한다.
- 포트폴리오 설명에서 실시간성, 의료 안전성, 정책 모델 한계를 과장하지 않는다.
