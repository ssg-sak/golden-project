# 골든 거버넌스 정책분석 방법론

이 문서는 대구 골든타임의 정책분석 엔진이 현재 공개 분석본을 만드는 절차를 설명한다. 분석 결과는 의료기관 신설 확정안이나 실제 환자 수용 예측이 아니라 현장조사 전 우선 검토 자료다.

## 1. 분석 계약

현재 2026.07.18 검증본은 다음 조건을 동시에 만족해야 한다.

| 항목 | 계약값 |
|---|---:|
| 행정동 | 150곳 |
| 기준 기관 | 25곳 |
| 소아 기준 기관 | 6곳 |
| 어르신 기준 기관 | 19곳 |
| 정책 후보 | 9곳 |
| 후보 추적 | 9곳 |
| 도로 경로 | 5,100건 |
| 성공 경로 | 5,100건 |
| 누락 경로 | 0건 |

기관·인구·후보·경로·분석 버전·해시 중 하나라도 일치하지 않으면 새 분석본을 공개하지 않는다.

## 2. 공간 입력과 수요 정의

- 행정동 경계와 중심점은 저장된 150개 GeoJSON을 사용한다.
- 인구는 2026.06 기준 0~9세와 65세 이상 주민등록인구를 사용한다.
- 전체 인구만 새로 조회됐다고 해서 연령별 인구를 비율로 추정하지 않는다.
- 소아 모드는 소아 야간·휴일 기준 기관 6곳, 어르신 모드는 대형·준종합 기준 기관 19곳을 공급 기준으로 구분한다.
- 행정동 중심점은 공간 비교를 위한 대표점이며 실제 환자 위치나 구급차 출발점이 아니다.

위경도는 WGS84(EPSG:4326)로 저장하고 미터 단위 공간 연산이 필요한 K-Means 단계에서는 EPSG:5179로 투영한다.

## 3. VDI 산출

### 후보 생성 전 거리 기반 지표

초기 공간분석은 최근접 기준 기관까지의 직선거리와 취약인구를 결합한다.

```text
거리 기반 VDI = ln(1 + 최근접 직선거리 km) × 취약인구
취약인구 = 0~9세 인구 + 65세 이상 인구
```

로그 변환은 원거리 극단값의 지배력을 완화하기 위한 분석 가정이다. 임상적 위험함수로 검증된 공식이 아니다.

### 현재 공개 도로 ETA 기반 지표

후보와 기준 기관의 도로 경로가 완성된 뒤 현재 공개 VDI를 다시 계산한다.

```text
현재 VDI = ln(1 + 일반 차량 ETA 분) × 취약인구
```

현재 경계값 13,261.43은 의료적 절대 임계값이 아니다. 150개 행정동을 높은 순으로 정렬했을 때 상위 25%를 우선 확인 대상으로 나눈 상대값이다.

## 4. 후보 생성과 민감도 분석

사각지대 수요점을 EPSG:5179로 투영한 뒤 K-Means로 1차 후보 중심을 만든다. 현재 후보 생성은 개별 수요점을 동일 가중치로 취급한다.

민감도 분석은 다음 조건을 조합해 **모드당 240개 조건**을 실행한다. 소아와 어르신을 분리해 계산하므로 전체 실행량은 480회이며, 서로 다른 두 모드의 결과를 한 시나리오로 합산하지 않는다.

- K: 2, 3, 4, 5
- 난수 시드: 0, 7, 21, 42, 100
- 최근접 기관 거리 상한: 없음, 10km, 15km, 20km
- 군위 처리: 포함, 별도 권역, 제외

3km 이내에 반복 등장하는 중심을 같은 후보권으로 묶고 반복률과 수요 규모로 안정 후보, 보류 후보, 별도 권역 후보를 구분한다. K-Means 중심은 실제 도로·부지·법규·예산을 고려한 최적 입지가 아니다.

## 5. 후보 근거 추적

안정 후보 9곳을 생성한 뒤 각 후보에 다음 근거를 연결한다.

- 후보 추가 전후 최근접 직선거리 변화
- 접근성 개선이 연결된 행정동
- 취약인구 가중 개선량
- 최근접 기존 기관
- 후보 분류와 민감도 반복률

후보 정본과 추적 파일의 모드·ID·위도·경도를 소수점 일곱 자리까지 비교한다. 9대9로 일치하지 않으면 다음 단계로 진행하지 않는다.

## 6. 실제 도로 ETA 행렬

카카오 일반 차량 경로를 사용해 다음 5,100개 조합을 구성한다.

```text
150개 행정동 × (25개 기준 기관 + 9개 후보) = 5,100개 경로
```

행렬은 입력 SHA-256과 경로 결과 SHA-256, 캐시 사용 수, 신규 조회 수, 좌표 보정 수를 기록한다. 현재 검증본은 5,100개 성공, 누락 0건이며 좌표 보정이 기록된 경로는 460건이다. 좌표 보정 거리의 경로별 최댓값 평균은 약 0.311km, 전체 최댓값은 약 0.573km다. 생성기는 허용 최댓값 0.75km를 넘는 보정 기록을 거부한다.

ETA는 수집시점의 일반 차량 경로다. 시간대별 교통, 구급차 우선통행, 출동 준비시간, 병원의 실제 수용 가능성을 보장하지 않는다.

## 7. p-median·MCLP 후보 조합 비교

분류된 후보군 안에서 시설 수 1~3개의 모든 조합을 비교한다.

- p-median: 취약인구 가중 평균 ETA를 최소화한다.
- MCLP 15분: 15분 이내 모델 커버 인구를 최대화한다.
- MCLP 30분: 30분 이내 모델 커버 인구를 최대화한다.

목적함수가 다르면 선택 조합도 달라질 수 있다. 결과는 후보군 내부의 정확해이며 대구 전역 모든 좌표를 탐색한 전역 최적해가 아니다.

현재 분석에는 전문의 수·MRI·CT 기반 인프라 패널티, 자원 보강 등급, 필요 인원·장비 수 산출을 사용하지 않는다.

## 8. 단일 분석본 생성과 실패 안전성

현재 파이프라인은 다음 순서로 실행된다.

```text
공간분석
→ 후보 민감도 분석
→ 안정 후보 9곳 생성
→ 후보 근거 추적
→ 실제 도로 ETA·p-median·MCLP
→ 후보 시각화
→ 단일 정책 분석본 검증·생성
```

통합 실행기는 시작 전에 공개 산출물, 도로 경로 캐시, 최종 병원 분석 입력을 임시 백업한다. 포착된 예외가 발생하면 해당 파일을 실행 전 상태로 복원하며, 백엔드 갱신 래퍼도 DB에서 내보내기 전 병원 입력 바이트를 별도로 복원한다. 새 결과는 구조·개수·버전·해시 검증을 모두 통과한 뒤에만 공개본으로 승격한다.

이 복구는 프로그램이 정상적으로 예외를 포착하는 실패를 대상으로 한다. 운영체제 강제 종료나 저장장치 장애까지 하나의 원자적 트랜잭션으로 보장하지는 않으므로, 운영 환경에서는 별도 파일시스템 스냅샷과 배포 아티팩트 보관이 추가로 필요하다.

## 9. 해석 한계

1. VDI는 현재 데이터 안의 상대 비교 지표이며 임상적 위험 확률이 아니다.
2. 후보는 현장조사와 전문가 검토의 우선순위이지 확정 입지가 아니다.
3. 모델 커버 인구는 실제 치료·수용 가능 인구가 아니다.
4. 정책분석 결과는 병상 준실시간 정보와 갱신 주기가 다르다.
5. 병원·행정동 원천의 정확한 개별 수집일과 실제 이송자료를 이용한 외부 타당성 검증이 남아 있다.
