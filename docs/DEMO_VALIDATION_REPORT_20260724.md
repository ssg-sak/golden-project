## 2. 변경 및 테스트 범위

### 변경 내용

- README 배포 주소를 `https://ssg-sak.github.io/golden-project/`로 교체
- README와 활성 화면의 병상 관련 `실시간` 표현을 `최근 조회` 계약으로 정합화
- README 배포 구성을 GitHub Pages·Render로 정정
- 존재하지 않는 Markdown 정책 보고서 링크를 현재 PDF로 교체
- 실제 배포 화면에서 시민·정책 대표 이미지를 캡처하고 README에 연결

### 테스트 포함 범위

- GitHub Pages 배포 상태와 HTTP 응답
- 프론트 단위 테스트, ESLint, TypeScript, 프로덕션 빌드
- 데스크톱 시민·정책 핵심 동선 연속 3회
- 모바일 시민 핵심 동선 연속 3회
- 최근 조회 병상값, 전화, 길찾기, 정책 핵심 지표
- 모바일 390×844 뷰포트의 가로 넘침과 정책 탭 비노출
- 브라우저 콘솔 오류와 공개 페이지 문구

### 테스트 제외 범위

- 전화·길찾기 링크를 통한 외부 앱의 최종 동작
- 실제 환자 진료·수용 가능 여부
- 신규 데이터 수집 또는 정책 분석 재실행

## 7. 실제 브라우저 반복 검증

### 데스크톱 핵심 동선

매회 `응급실 찾기`로 진입해 시민 화면을 새로 불러온 뒤 병원 상세를 열고, `정책·분석 모니터링`으로 전환해 핵심 지표를 확인했다.

| 반복 | 시민 화면 | 병원 상세·길찾기 | 정책 화면 | 150·25·5,100 지표 | 결과 |
|---:|---|---|---|---|---|
| 1 | PASS | PASS | PASS | PASS | PASS |
| 2 | PASS | PASS | PASS | PASS | PASS |
| 3 | PASS | PASS | PASS | PASS | PASS |

세 번 모두 최근 조회 일반응급실 가용병상 보고값이 표시된 `대구보훈병원` 상세를 열었다. 병상 보고값을 진료·수용 가능 여부로 단정하지 않는 안전 안내와 전화·길찾기 동선이 유지됐다.

### 모바일 핵심 동선

390×844 뷰포트에서 시민 화면을 새로 불러오고 병원 상세를 여는 절차를 3회 반복했다.

| 반복 | 시민 화면 | 병원 상세·길찾기 | 정책 탭 비노출 | 가로 넘침 | 결과 |
|---:|---|---|---|---|---|
| 1 | PASS | PASS | PASS | 없음 | PASS |
| 2 | PASS | PASS | PASS | 없음 | PASS |
| 3 | PASS | PASS | PASS | 없음 | PASS |

세 번 모두 문서 너비와 뷰포트 너비가 390px로 일치했다. 모바일에서는 시민 행동 동선을 우선하는 현재 설계에 따라 정책 탭이 노출되지 않았다.

### 콘솔 확인

앱 기능을 중단시키는 오류는 재현되지 않았다. Chrome 확장 메시지 채널이 응답 전에 닫혔다는 동일 오류 메시지 2건이 기록됐으나, 시민·정책 화면 동작과 데이터 표시에 영향은 없었다. 메시지 특성상 브라우저 확장 연동에서 발생한 것으로 보이지만, 이 보고서에서는 원인을 확정하지 않는다.
